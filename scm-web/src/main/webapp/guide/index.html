Markup Guide

프로젝트 환경

  • 아래 종류는 마크업 작업시 기본적인 가이드 관련 내용입니다.
  • HTML 또는 개발 작업시 지켜야 할 항목들 입니다.
  • OS 지원 / 문서타입 / 기본 해상도 / 인코딩 은 기획단계에서 나오기 때문에 꼭 확인해야 합니다.

OS 지원

WINDOW 10 이상

문서타입

<!DOCTYPE HTML>, html5, css3

인코딩

UTF-8

브라우저

기본적으로 크로스 브라우징은 익스플로러 10이상, 크롬을 맞추며, 최신브라우저를 기준으로 한다.

기본 해상도

1280px X 720px (디자인 상황에 따라 달라질수 있음)

CSS 수정 및 설정

  • inline style을 되도록 허용 하지 않으며, from 태그에 관해서만 허용한다. (width, height 값만 허용하도록 한다)
  • 개발시 수정이 필요한 CSS가 있을 경우 반드시 작업자에게 요청.

UI 및 컴포넌트

  • 본 프로잭트의 web application은 Bootstrap v3.3.7 라이브러리를 사용합니다.
  • Bootstrap v3.3.7의 API는 https://getbootstrap.com/ 를 참고합니다.

차트 설정

  • 본 프로잭트의 Chart는 amcharts(https://www.amcharts.com/)를 사용합니다.
  • amcharts의 API는 https://www.amcharts.com/docs/v4/를 참고합니다.
  • amcharts는 웹에서 커스터마이징하여 코드를 작성해볼 수 있는 Editor를 제공해주는데 https://editor4.amcharts.com/am4editor/#/am4editor/design 를 접속하여 유용합니다.
